# Lab 2 首頁翻新任務提示

這份文件用來銜接 `取材任務提示.md` 的下一階段。請在完成首頁素材盤點後，再把這份提示交給 Codex 執行，目標是依照已整理素材產出一版可展示、可閱讀、可維護的靜態公司首頁。

## 任務目標

請依照 `labs/lab 2/` 中已整理好的素材，完成公司首頁翻新版靜態頁面。

本任務要實作新版首頁，不要重新取材，也不要把素材文件大段複製到頁面中。請根據素材整理結果，挑選可確認、可沿用的內容，建立一版結構清楚、適合公司形象展示的首頁。

## 執行前確認

開始前請先讀取以下文件：

```text
labs/lab 2/首頁素材盤點.md
labs/lab 2/首頁文案整理.md
labs/lab 2/圖片素材清單.md
labs/lab 2/來源紀錄.md
labs/lab 2/ModernCSS.md
labs/lab 2/javascript.md
```

如果素材盤點文件尚未建立，請先停止並回報缺少哪些素材檔，不要憑空產生公司事實、服務項目、客戶名稱或專案成果。

## 技術選型

請使用：

- HTML5
- 現代 CSS
- Vanilla JavaScript

請不要使用：

- React
- Angular
- Vue
- Tailwind
- Bootstrap
- 需要 `npm install` 或 build step 的工具

本 lab 的重點是讓初學者看懂 Codex 如何從素材整理進入靜態首頁實作。首頁大多是靜態內容瀏覽，使用原生 HTML、CSS、JavaScript 就足夠，也比較容易在 live demo 中檢查與講解。

### HTML 要求

- 使用語意化標籤，例如 `header`、`nav`、`main`、`section`、`footer`。
- 主要內容必須直接存在於 HTML 中，不要依賴 JavaScript 動態產生。
- 導覽、標題、段落、CTA、聯絡資訊要有清楚層級。
- 圖片需提供合理的 `alt`，不要使用空泛文字。

### CSS 要求

- CSS 不要寫成「CSS3」或「CSS4」。現在 CSS 是由多個 modules 持續演進，請以現代瀏覽器穩定支援的 CSS 能力為準。
- 可使用 CSS variables、Flexbox、Grid、media queries、`clamp()`、`object-fit`、`scroll-padding`、`:where()` 等現代 CSS。
- 可參考 `ModernCSS.md`，但只挑選穩定、適合靜態首頁、容易向初學者解釋的寫法。
- 不要使用過於實驗性、瀏覽器支援不穩定、或需要大量 fallback 的 CSS。
- 版面要支援手機、平板、桌面，不可出現文字重疊或水平捲動。

### JavaScript 要求

- JavaScript 只處理必要互動，例如手機選單、導覽狀態、錨點捲動、簡單 accordion。
- 可參考 `javascript.md` 的語法與 DOM 操作，但不要引入複雜架構或不必要的抽象。
- 不要用 JavaScript 儲存或產生主要公司文案。
- 若瀏覽器不支援 JavaScript，主要內容仍應可閱讀。

## 參考資料使用規則

- `ModernCSS.md` 是 CSS 寫法參考，不是必須全用的功能清單。
- `javascript.md` 是 JavaScript 語法與 DOM 操作參考，不是要求建立大型前端程式。
- 不要把大型參考文件內容複製到成果頁。
- 若參考文件中有實驗性或不適合本首頁的範例，請略過。
- 優先使用素材盤點與文案整理中的第一方內容。
- OCR、推測、外部參考資料必須保留標註，不要改寫成確定事實。

## 建議頁面結構

新版首頁至少包含：

- Hero 首屏：公司名稱、核心主張、主要 CTA。
- 公司定位：說明公司主要服務與價值。
- 服務項目：整理素材中可確認的服務內容。
- 解決方案或專案實績：只使用素材中可追溯的內容。
- 產業應用或客戶情境：如果素材不足，請改成較保守的服務情境描述。
- 聯絡 CTA：引導使用者聯絡或了解更多。
- Footer：公司基本資訊、導覽連結、聯絡資訊。

請依照素材實際內容調整區塊名稱，不要為了湊區塊而捏造不存在的案例或客戶。

## 輸出位置

請建立以下目錄與檔案：

```text
labs/lab 2/homepage-refresh/index.html
labs/lab 2/homepage-refresh/styles.css
labs/lab 2/homepage-refresh/script.js
labs/lab 2/homepage-refresh/assets/
```

若有沿用圖片，請放在 `assets/`，並在 HTML 或 CSS 中使用相對路徑引用。

## 驗收標準

完成後請自我檢查：

- 直接開啟 `index.html` 可正常瀏覽，不需要 `npm install` 或 build。
- 手機、平板、桌面版面正常，沒有文字重疊或水平捲動。
- 導覽、手機選單、CTA、錨點連結可操作。
- 主要內容存在於 HTML。
- 圖片都有合理 `alt`。
- 不確定來源、OCR、推測、外部參考仍依 lab 2 規則標註。
- 沒有把第三方徵才網站內容誤標成首頁原文。
- 沒有提交憑證、cookie、追蹤資料或敏感資訊。
- 沒有把 `ModernCSS.md` 或 `javascript.md` 的大量內容複製進成果頁。
